# INTELIAR MASTER SPECIFICATION
## Capítulo 18 — Discover Engine

**Versión:** 1.0
**Estado:** Core Experience
**Prioridad:** Máxima

---

## Objetivo

Definir el proceso mediante el cual Inteliar descubre, comprende y diseña una solución **antes** de construirla.

El Discover Engine es la puerta de entrada de toda empresa. No importa si el resultado será un SaaS, un ERP, un CRM, un sistema interno, una automatización o un AI Pod. Todo comienza aquí.

## Filosofía

Las empresas no saben qué software necesitan. Saben qué problemas tienen. El trabajo de Inteliar no consiste en vender módulos: consiste en comprender esos problemas.

---

## La Primera Conversación

No es un formulario. No es un onboarding. No es un wizard. Es una conversación.

```
Hola. Soy The Architect.
Quiero entender cómo funciona tu empresa.
No voy a venderte nada. Primero quiero conocerte.
```

### Las ocho preguntas

The Architect intenta responder únicamente ocho preguntas:

1. ¿Qué hace esta empresa?
2. ¿Cómo gana dinero?
3. ¿Cómo trabaja hoy?
4. ¿Qué herramientas utiliza?
5. ¿Qué duele?
6. ¿Qué quiere mejorar?
7. ¿Qué restricciones existen?
8. ¿Cómo debería verse dentro de cinco años?

---

## El resultado nunca es software

El resultado es **conocimiento**. Cuando termina, se generan automáticamente:

```
Business Profile
Digital Twin
Capability Map
Risk Analysis
Automation Opportunities
AI Opportunities
Technical Assessment
Implementation Roadmap
```

Todavía no existe código.

## El Deliverable

Cuando termina Discover, el cliente recibe un documento (no una cotización):

```
Estado Actual → Objetivo → Arquitectura Propuesta → Roadmap →
Estimación → Beneficios Esperados → Próximos Pasos
```

## Conversión — tres caminos

- **Camino 1 — Business:** el equipo Inteliar implementa.
- **Camino 2 — Partner:** un Partner certificado implementa.
- **Camino 3 — Builder:** el cliente o su equipo construyen.

## El verdadero producto

Discover no existe para vender. Existe para generar claridad. Aunque la empresa nunca compre, debería terminar entendiendo mejor su negocio.

---

## Integraciones

- **Builder:** si el cliente construye, Builder ya tiene Digital Twin, Capability Map, Roadmap, Estimaciones, Arquitectura. No comienza desde cero.
- **Business:** si contrata, el Arquitecto Humano recibe todo. No vuelve a preguntar.
- **Marketplace:** The Architect propone Templates, Partners, Agentes, Industry Packs. Todo personalizado.
- **Knowledge:** cada Discover aumenta el Knowledge Engine, incluso si nunca existe un proyecto.

## Discover como Servicio

Una empresa podría contratar únicamente el proceso de descubrimiento, sin construir software. Eso ya genera muchísimo valor.

```
Empresa → Discover Engine → The Architect → Knowledge Engine →
Digital Twin → Capability Map → Roadmap → Business Decision →
[Builder | Business | Partner]
```

---

## 🚨 ADR-024
Todo proyecto dentro de Inteliar comienza con un proceso de Discover. No existen excepciones. Sin comprender el negocio, no existe arquitectura.

## ⭐ ADR-025 — Discover First
En Inteliar nunca se vende software primero. Primero se vende comprensión. Después arquitectura. Después transformación. Y recién al final aparece el software.

## 📌 La idea más grande — el ciclo de vida

```
INTELIAR
1. Discover  — Comprender la empresa
   ↓
2. Design    — Diseñar la solución
   ↓
3. Build     — Construir la plataforma
   ↓
4. Operate   — Operar el negocio
   ↓
5. Evolve    — Mejorar continuamente
```

No son cinco productos. Son las cinco etapas del ciclo de vida de una empresa dentro de Inteliar.

El mensaje principal de la compañía ya no es *"Construí tu SaaS"*, sino:

> **"Diseñamos, construimos y hacemos evolucionar el sistema operativo de tu empresa."**
